# Tailwind v4 마이그레이션 체크리스트: 5가지 핵심 변경점

---

## 1. 요약 (3줄)
- Tailwind CSS v4로 업그레이드할 때 반드시 확인해야 할 5가지를 정리했습니다
- @config 지시자가 @import 기반으로 변경
- 색상 시스템이 OKLCH 기반으로 전환

## 2. 왜 중요한가

Tailwind v4 마이그레이션 체크리스트: 5가지 핵심 변경점은(는) 현재 개발 트렌드에서 핵심적인 위치를 차지하고 있습니다. 기존 워크플로우를 크게 개선할 수 있는 잠재력을 가지고 있으며, 특히 반복적인 작업을 줄이는 데 효과적입니다.

**실제 시나리오**: 새 프로젝트를 셋업하거나 기존 코드를 리팩토링할 때, 이 방법을 적용하면 초기 세팅 시간을 절반 이하로 줄일 수 있습니다.

**언제 쓰면 좋을까?** 빠른 프로토타이핑, 반복 작업 자동화, 팀 온보딩 시 특히 유용합니다. 반대로, 보안이 극도로 중요한 프로덕션 환경이나 레거시 시스템과의 깊은 통합이 필요한 경우에는 신중하게 적용해야 합니다.

## 3. 바로 써먹기

- [ ] @config 지시자가 @import 기반으로 변경
  > _💡 기본 환경을 올바르게 구성해야 이후 단계에서 오류가 줄어듭니다._
- [ ] 색상 시스템이 OKLCH 기반으로 전환
  > _💡 핵심 의존성을 먼저 확인하면 나중에 디버깅 시간을 절약합니다._
- [ ] 플러그인 API가 CSS-first로 변경
  > _💡 이 단계를 건너뛰면 런타임 에러가 발생할 수 있습니다._
- [ ] content 설정이 자동 감지로 전환
  > _💡 설정 검증을 통해 배포 시 문제를 사전에 방지합니다._
- [ ] darkMode 설정 방식 변경
  > _💡 테스트를 먼저 실행하면 회귀 버그를 즉시 발견할 수 있습니다._
- [ ] npx @tailwindcss/upgrade 실행
  > _💡 문서화를 해두면 팀원 온보딩이 빨라집니다._
- [ ] tailwind.config.js → CSS @theme으로 이전
  > _💡 최종 점검으로 프로덕션 안정성을 확보합니다._
- [ ] 커스텀 플러그인 있으면 CSS 방식으로 재작성
  > _💡 모니터링 설정으로 장기적인 유지보수가 수월해집니다._

## 4. 예시

```bash
# 1) 프로젝트 초기화
mkdir my-project && cd my-project
npm init -y

# 2) 필수 패키지 설치
npm install express dotenv

# 3) 실행 확인
node index.js
# → http://localhost:3000 에서 확인
```

> 위 예시는 일반적인 패턴이며, 실제 프로젝트에 맞게 수정하세요.
